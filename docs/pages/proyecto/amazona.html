Acerca del proyecto

El proyecto es una tienda online, inspiarada en Amazon, que ha sido creada empleando el Stack MEAN.

Del lado del frontend se ha utilizado Angular 12, framework del lado del cliente del cual he usado:

  • Componentes para construir la página, usando elementos reutilizables y las vistas de cada página.
  • Guards para verificar el acceso a determinadas rutas de la app.
  • Interfaces para definir estructuras de datos como, por ejemplo, los productos.
  • Pipes para normalizar y dar formato a los datos que se van a mostrar al usuario desde la misma estructura HTML del proyecto.
  • Servicios para comunicar la app con la API-REST y/o comunicar los distintos componentes de la app entre sí.
  • Formularios reactivos para validar en tiempo real los campos del formulario.
  • Cookies para almacenar datos como los JSON Web Token que nos envía el servidor tras autenticarnos o refrescar el token tras expirar la cookie.
  • Angular Material librería para utilizar componentes como sugerencias de búsqueda, menús, diálogos para confirmar acciones, etc.

Del lado del backend se han utilizado Express y MongoDB:

  • Enrutadores para crear los endpoints, de la API-REST, que serán atacados desde el cliente usando peticiones HTTP: POST, GET, PUT, DELETE.
  • CRUD creado con Mongoose (ODM de MongoDB) para las distintas acciones con la base de datos: Create, Read, Update & Delete.
  • JWT (JSON Web Token) tecnología empleada para validar las acciones del usuario. Cuando un usuario envíe sus credenciales al servicio de autenticación, al iniciar sesión en su cuenta, el servidor le mandará al cliente un token y un token de refresco tras comprobar la autenticidad del mismo. Cada vez que el usuario realice una acción que modifique su cuenta como, por ejemplo:
    • Añadir un producto al carrito.
    • Añadir, editar o eliminar una dirección.
    • Añadir o eliminar una tarjeta de crédito.
    • Realizar un pedido.

    Deberá enviar al servidor el token que le fue entregado al iniciar sesión. El servidor se encargrá de validar el token comprobando que haya sido firmado por él mismo (haciendo uso de su clave secreta) y que no haya vencido su caducidad, para así poder realizar la acción requerida por el usuario.

    En caso de que la cookie donde esté almacenado el token haya caducado, si la app conserva el token de refresco, previamente a la acción que requiere el token se mandará al servicio de autenticación el token de refresco.

    En caso de que el token de refresco haya sido firmado por el servidor, que no haya vencido su fecha límite y que sea el mismo token de refresco que se le mandó al usuario cuyo ID coincida con el ID almacenado en el propio token, el servidor generá un nuevo token y otro de refresco y se los enviará al cliente, permitiendo realizar la acción solicitada sin que el usuario tenga que volver a iniciar sesión para obtener un nuevo token.

Stack MEAN

Librerías

Este proyecto ha sido creado usando el Stack MEAN: (MongoDB, Express, Angular y Node).

Amazona está creada con Angular 12, apoyada en un conjunto de librerías como: Angular Material y Bootstrap 5.

Para su funcionamiento se conecta a una API-REST creada con el framework Express y utilizando MongoDB, como base de datos no relacional, y el ODM Mongoose para la creación del CRUD. Al igual que en la app, del lado del backend se han empleado otras librerías como: NodeMailer, JWT (JSON Web Token), DotEnv, entre otras...

¿Qué puede hacer?

Realizar compras de los productos que ofrece la tienda

  • Buscar productos: por sugerencias (etiquetas ordenadas de mayor a menor número de búsquedas) o por nombre.
  • Ver información de un producto y ver sugerencias de otros productos de la misma categoría
  • Hacer zoom en las imágenes de un producto, cambiar de imagen si posee más de una, añadir el producto a la cesta...
  • Editar productos de la cesta: incrementar/decrementar items de un producto, vaciar cesta, etc.
  • Registrar una cuenta y posteriormente activarla con el enlace enviado al correo registrado.
  • Añadir, editar o eliminar direcciones para el envío del pedido.
  • Añadir o eliminar tarjetas de crédito para poder realizar el pago de un pedido.
  • Ver lista de pedidos realizados, así como un pedido en detalle.
  • Procesar un pedido.
